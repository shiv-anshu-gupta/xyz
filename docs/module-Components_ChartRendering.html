    Computed Channel Chart Renderer

    Orchestrates rendering of computed channels by delegating to shared renderers.

    What This Module Does

    StepAction
    1Reads computed channels from cfg.computedChannels + data.computedData
    2Separates analog vs digital via madeFrom field
    3Groups channels by their group field
    4Calls renderSingleAnalogChart() for analog types
    5Calls renderSingleDigitalChart() for digital types

    Design Philosophy

    • Unified Rendering — Uses same renderers as COMTRADE data
    • Computed-Only Mode — Passes channelIndices=[] to signal computed-only
    • Group-Based — Each channel's group field determines chart ownership
    • Type-Aware — Routes to correct renderer based on madeFrom

    Ownership Rule

    Computed channels with group matching an existing analog/digital group are rendered in that group's chart. Unmatched groups get standalone charts.

    Data Flow

    1. cfg.computedChannels[i] + data.computedData[i]
    2. ↓ buildComputedChannelObjects()
    3. ↓ separateByType() + groupByGroup()
    4. ↓ analogGroups / digitalGroups
    5. ↓ renderSingleAnalogChart() / renderSingleDigitalChart()
    6. Charts with computed channels only

    • Main Chart Rendering Orchestrator

      Entry point for chart visualization after COMTRADE file parsing. Orchestrates rendering of all chart types: analog, digital, and computed.

      Responsibilities

      TaskDescription
      CleanupClear existing charts and metadata
      Drag & DropEnable chart reordering
      CoordinationRender analog → digital → computed
      SubscriptionsManage chart metadata updates
      Delta CalcTrigger calculations for vertical markers

      Rendering Pipeline

      1. destroyCharts() + clearContainer()
      2. analyzeGroupsAndPublishMaxYAxes()
      3. Phase 1: Analog Charts (one per group)
        ↳ Include computed channels with matching group
      4. Phase 2: Digital Charts (one per group)
      5. Phase 3: Standalone Computed Channels
      6. Setup delta calculations

      Ownership Rule

      Computed channels with group matching an analog group render inside that chart. Only unmatched groups get standalone charts.

    • module:components/renderSingleAnalogChart

    • module:utils/autoGroupChannels

renderComtradeCharts(

  verticalLinesX,   // Vertical lines state
  createState,      // State factory

    • Digital Channel Chart Renderer

      Renders binary/digital channel waveforms as stepped graphs showing HIGH (1) and LOW (0) states over time. Each digital group gets its own chart with channels stacked vertically. Uses a specialized fill plugin for clear state visualization.

      Design Philosophy

      PrincipleDescription
      Group RenderingOne chart per digital group with stacked channels
      Smart FilteringSkips channels with no HIGH values (all zeros)
      Fill PluginVisual state indication via filled regions
      Drag SupportDrag bars for chart reordering

      Key Features

      • Binary Display — Shows 0/1 states as stepped waveforms
      • Stacked Layout — Multiple channels per chart with vertical offset
      • Empty Channel Skip — Filters out channels with no state changes
      • Fill Plugin — Visual fills for HIGH state regions
      • Vertical Lines — Supports measurement markers
      • Drag Bar — Reorderable via drag handle

      Data Validation

      CheckAction
      Channel has all zerosSkip rendering (no meaningful data)
      Group has no visible channelsSkip entire chart
      Invalid data arrayLog warning and skip
    See:
    • module:plugins/digitalFillPlugin - Visual fill plugin
    • module:components/renderComtradeCharts - Parent orchestrator
    • module:utils/digitalChannelUtils - Digital channel utilities
    Example
    // Render digital charts for a specific group
renderDigitalCharts(
  cfg,              // COMTRADE config
  data,             // COMTRADE data
  chartsContainer,  // DOM container
  charts,           // Chart instances array
  verticalLinesX,   // Vertical lines state
  channelState,     // Channel state object
  { name: "G1", indices: [0, 1, 2] }  // Optional group

    • Single Analog Chart Renderer

      Core building block for analog chart visualization in COMTRADE viewers. Renders a SINGLE uPlot instance for one channel group.

      Design Philosophy

      PrincipleDescription
      Single ResponsibilityOne function → one chart
      DRYUses shared utilities to avoid duplication
      Ownership RuleComputed channels render with their parent group
      ConsistencySame interface as renderSingleDigitalChart

      Key Features

      • Group-based rendering — Channels grouped by user or auto-grouping
      • Computed channel integration — Ownership rule merges computed into parent
      • Auto data validation — Filters invalid/empty channels
      • Shared event handlers — Click for vertical lines, hover for tooltip
      • Responsive charts — ResizeObserver auto-adjusts

      Pipeline

      1. Input: groupId, channelIndices, cfg, data
      2. Validate Inputs → Invalid? Return null
      3. Build chartData Array
      4. Extract Labels / Colors / Units
      5. createDragBar() → createChartContainer()
      6. createChartOptions() → initUPlotChart()
      7. Attach Metadata → attachChartEventHandlers()
      8. Return chart instance
    See:
    • module:components/renderSingleDigitalChart - Digital equivalent
    • module:components/renderComtradeCharts - Orchestrator
    • module:utils/autoGroupChannels - Grouping logic
    Example
    import { renderSingleAnalogChart } from "./renderSingleAnalogChart.js";

renderSingleAnalogChart(
  "G0",              // Group ID
  [0, 1, 2],         // Channel indices (IA, IB, IC)
  0,                 // Chart index
  cfg, data,         // COMTRADE data
  chartsContainer,   // DOM container
  charts,            // Charts array
  verticalLinesX,    // Vertical lines state
  { channelState, maxYAxes: 3, computedChannels: [] }
);

    • Single Digital Chart Renderer

      Visualizes binary/boolean signals from protection relays and power quality meters. Renders a SINGLE uPlot instance for one digital channel group.

      Design Philosophy

      PrincipleDescription
      Single ResponsibilityOne function → one chart
      DRYUses shared utilities to avoid duplication
      Stacked DisplayColor-coded HIGH/LOW states
      ConsistencySame interface as renderSingleAnalogChart

      Key Features

      • Group-based rendering — Channels grouped by user or auto-grouping
      • Stacked display — Configurable DigChannelOffset between channels
      • Color-coded fill — HIGH states shown as filled rectangles
      • Auto filtering — Channels with no HIGH values are hidden
      • Computed support — Can render computed digital channels
      • Responsive — ResizeObserver auto-adjusts

      Operating Modes

      ModechannelIndicescomputedChannelsResult
      COMTRADE[0,1,2][]Renders COMTRADE channels
      Computed-only[][...]Renders only computed
      Mixed[0,1,2][...]Both in same chart

      Digital Signal Display

      • HIGH (1) — Filled rectangles
      • LOW (0) — Baseline
      • Stacked — Vertical offset between channels
    See:
    • module:components/renderSingleAnalogChart - Analog equivalent
    • module:components/renderComtradeCharts - Orchestrator
    • module:plugins/digitalFillPlugin - Fill plugin
    Example
    import { renderSingleDigitalChart } from "./renderSingleDigitalChart.js";

renderSingleDigitalChart(
  "G0",              // Group ID
  [0, 1, 2, 3],      // Channel indices (TRIP, CLOSE, 52A, 52B)
  1,                 // Chart index
  cfg, data,         // COMTRADE data
  chartsContainer,   // DOM container
  charts,            // Charts array
  verticalLinesX,    // Vertical lines state
  { channelState }   // Options
);
