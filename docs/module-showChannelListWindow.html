Module

showChannelListWindow

Opens a child popup window for interactive channel editor with Tabulator grid interface. Allows users to edit channel metadata (names, colors, groups, properties) with automatic updates reflected back to the main application. Key Features: - Dual popup windows (callback-based and postMessage-based) - Tabulator grid for tabular channel editing - Tailwind CSS for responsive UI - Separate analog and digital channel sections - Automatic group assignment using autoGroupChannels - Color picker integration for channel colors - Undo/Redo history support - PDF export of channel list - Add/Delete row functionality - PostMessage for cross-window communication Channel State Synchronization: - Child window serializes channelState to JSON for safety - createChannelList runs in child window context - Changes posted via window.opener.postMessage() - Parent listens and updates channelState (createState) - State changes trigger chartManager subscribers - Charts automatically update with new colors/labels Dependencies: - ChannelList.js: Channel editor Tabulator grid - autoGroupChannels.js: Intelligent channel grouping by unit/phase - Tabulator: External grid library (CDN loaded) - Tailwind CSS: Styling (CDN loaded)

import { showChannelListWindow } from './components/showChannelListWindow.js';

const channel Popup = showChannelListWindow(

    console.log(`Moved ${type} from ${fromIdx} to ${toIdx}`);
  },
  (type, idx, color) => {              // onChannelColorChange callback
    console.log(`Changed ${type}[${idx}] color to ${color}`);
  },
  charts                               // Charts array for digital color mapping
);

// Popup will open with Tabulator grid showing all channels
// User edits are automatically synced back to channelState

# inner showChannelListWindow(channelState, onChannelDropopt, onChannelColorChangeopt, chartsopt) → {Window|undefined}

Open a Channel List popup and initialize the child UI. This function prepares the child window (loads Tailwind + Tabulator) and injects the ChannelList UI into the popup. It serializes the current `channelState` (analog/digital arrays) and passes them to the child. The child runs `createChannelList` inside its own context and uses `window.opener.postMessage(...)` to notify the parent of user-driven changes (color/name/add/delete). The parent listens for these messages and updates `channelState` (createState), which in turn notifies chart subscribers. Workflow: 1. Create popup window with Tailwind + Tabulator CSS 2. Serialize current channelState to JSON 3. Auto-group analog channels by unit/phase using autoGroupChannels 4. Inject module script into popup that: a. Imports createChannelList b. Deserializes config c. Creates Tabulator grid d. Listens for user edits e. Posts changes via postMessage 5. Return popup window reference Note: the child may also call the `onChannelColorChange` or `onChannelDrop` callbacks directly if they are supplied, keeping compatibility with the existing callback-based integration.
