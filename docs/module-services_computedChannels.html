    Data Preparation for Web Worker

    Prepares COMTRADE data for zero-copy transfer to the computation Web Worker.

    Key Optimization

    Only converts channels that are actually used in the expression! Reduces memory transfer from ~300MB (all 599 channels) to ~1-5MB (only used).

    Process

    StepAction
    1Extract analog/digital arrays from globalData
    2Parse expression for used channel names (IA, IB, a0, d5...)
    3Convert only used channels to Float64Array
    4Build transferable objects array for postMessage()
    5Serialize channel metadata for worker scope

    ArrayBuffer Transfer

    ArrayBuffers are transferred (not copied) to Web Workers:

    • Zero-copy operation — moves ownership of memory
    • Original buffer is detached after transfer
    • Cannot use original buffer after postMessage()

    • LaTeX to Math.js Converter

      Converts LaTeX expressions from MathLive editor to math.js compatible format.

      Why Conversion is Needed

      MathLive (LaTeX)Math.js (JavaScript)
      \sqrt{x}sqrt(x)
      I_{A}IA
      \frac{a}{b}(a)/(b)
      x^{2}x^(2)
      \cdot*

      Supported LaTeX Constructs

      LaTeXConverted To
      I_{A} (subscript)IA
      \sqrt{x}sqrt(x)
      \frac{a}{b}(a)/(b)
      \operatorname{RMS}(x)sqrt(mean((x)^2))
      \cdot, \times*
      \left\lvert x \right\rvertabs(x)
      x^{2}x^(2)
      \left( \right)( )
    See:
    • module:services/computedChannels - Uses this converter
    • module:components/EquationEvaluatorInChannelList - MathLive source
    Example
    import { convertLatexToMathJs } from "./expressionConversion.js";

convertLatexToMathJs("\\sqrt{I_{A}^2+I_{B}^2+I_{C}^2}");
// Returns: "sqrt(IA^(2)+IB^(2)+IC^(2))"

convertLatexToMathJs("\\frac{V_{A}}{I_{A}}");
// Returns: "(VA)/(IA)"

    • Computed Channel Evaluation Orchestrator

      Main entry point for computed channel evaluation. Handles the entire flow from LaTeX input to chart rendering.

      Pipeline Overview

      User Input (LaTeX) → Validation → Conversion → Data Prep → Web Worker → Results → State → Events

      Responsibilities

      StepModuleAction
      1validatorsValidate expression & global data
      2expressionConversionConvert LaTeX → math.js
      3dataPreparationPrepare zero-copy ArrayBuffers
      4workerManagementCreate & manage Web Worker
      5resultProcessingProcess results & calc stats
      6stateUpdateUpdate cfg, data, channelState
      7eventHandlingDispatch UI update events

      Performance Optimizations

      • Web Worker — Non-blocking computation
      • Transferable ArrayBuffers — Zero-copy data transfer
      • Selective Transfer — Only channels used in expression (not all 599+)
      • Progress Reporting — For long computations
    See:
    • module:workers/computedChannelWorker - Worker implementation
    • module:components/renderComputedChart - Chart rendering
    Example
    import { handleComputedChannelEvaluation } from "./services/computedChannels/index.js";

window.addEventListener("message", async (e) => {
  if (e.data.type === "evaluateComputedChannel") {
    await handleComputedChannelEvaluation(e.data.payload);
  }
});

    • Worker Results Processing

      Processes results from the computation Web Worker. Handles ArrayBuffer conversion, statistics, and metadata building.

      Data Flow

      1. Worker resultsBuffer (ArrayBuffer)
      2. ↓ convertResultsToArray()
      3. JavaScript Array
      4. ↓ calculateStatistics()
      5. Stats: { min, max, mean, count }
      6. ↓ buildChannelData()
      7. Output: { metadata, values }

      Output Format

      Matches the analog/digital data pattern:

      OutputDestinationAnalog Equivalent
      metadatacfg.computedChannels[i]cfg.analogChannels
      valuesdata.computedData[i]data.analogData
    See:
    • module:services/computedChannels - Main orchestrator
    • module:services/computedChannels/stateUpdate - Consumes output
    Example
    const results = convertResultsToArray(resultsBuffer);
const stats = calculateStatistics(results);
const channelData = buildChannelData(results, expr, mathExpr, unit, stats, "MyChannel");

saveToGlobalData(channelResult);  // Updates data.computedData

updateStateStore(channelResult);  // Updates reactive stores

    • Computed Channel Input Validators

      Validates all inputs before computed channel evaluation begins. Provides early failure with descriptive error messages.

      Validation Chain

      StepFunctionPurpose
      1validateExpressionPayloadCheck expression exists, extract name
      2validateGlobalDataEnsure cfg/data are available
      3validateSampleDataVerify analog data has samples
      4validateExpressionSyntaxTest math.js can compile

      Name Extraction

      Supports named expressions:

      • Input: "I_RMS = sqrt(IA^2 + IB^2 + IC^2)"
      • Name: "I_RMS"
      • Expression: "sqrt(IA^2 + IB^2 + IC^2)"
    See:
    • module:services/computedChannels - Main orchestrator
    Example
    import { validateExpressionPayload, validateGlobalData } from "./validators.js";

const result1 = validateExpressionPayload({ expression: "IA + IB" });
if (!result1.valid) return console.error(result1.error);

const result2 = validateGlobalData(window.globalCfg, window.globalData);
if (!result2.valid) return console.error(result2.error);

    • Web Worker Lifecycle Management

      Creates and manages Web Workers for computed channel evaluation.

      Web Worker Benefits

      • Non-blocking — UI stays responsive during computation
      • Parallel processing — Runs on separate thread
      • Zero-copy transfer — ArrayBuffers moved, not copied

      Worker Lifecycle

      1. createComputedChannelWorker() → Create
      2. buildWorkerMessageHandler() → Attach handlers
      3. buildWorkerErrorHandler() → Attach error handler
      4. sendTaskToWorker() → Send data
      5. Auto-terminate on complete/error

      Message Types from Worker

      TypePayloadDescription
      progress{ percent, processed, total }Progress update
      complete{ resultsBuffer, sampleCount }Success
      error{ message }Failure
    See:
    • module:services/computedChannels - Main orchestrator
    • module:workers/computedChannelWorker - Worker implementation
    Example
    const worker = createComputedChannelWorker();
worker.onmessage = buildWorkerMessageHandler(worker, startTime, ...);
worker.onerror = buildWorkerErrorHandler(worker, onError);
sendTaskToWorker(worker, task, transferableObjects);
