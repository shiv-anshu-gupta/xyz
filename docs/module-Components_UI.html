    • Lightweight Debug Panel

      A minimalistic debug logging panel that displays tagged log entries in a floating sidebar overlay. Designed for development-time debugging with performance safeguards to prevent UI freezes from excessive DOM operations.

      Design Philosophy

      PrincipleDescription
      Disable by Default_enabled = false prevents all DOM work in production
      Throttled Logging300ms throttle prevents spam from rapid repeated logs
      Tag Filtering_importantPrefixes filters logs to relevant categories
      Instant BailoutWhen disabled, log() returns immediately with no overhead

      Key Features

      • Toggle Enable — Set _enabled = true to activate (WARNING: causes freezes)

    • Progress Indicator Component

      A lightweight, fixed-position progress bar that displays computation progress for Web Worker operations such as COMTRADE file parsing, computed channel evaluation, and data processing tasks.

      Design Philosophy

      PrincipleDescription
      Reactive StateUses createState for automatic UI updates on property changes
      Non-BlockingFixed position at top of viewport, doesn't affect page layout
      Smooth AnimationCSS transitions for width changes (0.3s ease-out)
      Gradient StylingPurple gradient with glow effect for visual prominence
      Zero ConfigSelf-contained styling via inline CSS

      Key Features

      • Global State — Exported progressState accessible from any module
      • Auto-Hide — Fades out when visible=false or percent=0
      • Percentage Clamping — Values automatically clamped to 0-100 range
      • Message Support — Optional status message for detailed feedback
      • Glow Effect — Box shadow creates illuminated appearance

      State Properties

      PropertyTypeDefaultDescription
      visiblebooleanfalseControls opacity (1 or 0)
      percentnumber0Progress value 0-100
      messagestring"Processing..."Status text

      Usage Pattern

      1. Import and call createProgressBar() to get DOM element
      2. Append element to document.body
      3. Call showProgress(percent, message) to display
      4. Call updateProgress(percent) during operation
      5. Call hideProgress() when complete
    See:
    • module:components/createState - Reactive state management
    • module:workers/comtradeWorker - File parsing worker
    Examples
    // Setup progress bar
import { createProgressBar, showProgress, updateProgress, hideProgress } from './ProgressBar.js';

document.body.appendChild(createProgressBar());

// Show during file parsing
showProgress(0, "Loading COMTRADE file...");

// Update from worker progress events
worker.onmessage = (e) => {
  if (e.data.type === 'progress') {
    updateProgress(e.data.percent, e.data.message);
  }
  if (e.data.type === 'complete') {
    hideProgress();
  }
};
    // Direct state manipulation
import { progressState } from './ProgressBar.js';

    (inner) createDragBar(group, cfg, channelState) → {HTMLDivElement}

    Description:

    • Create a draggable color bar displaying channel identities for a chart.

      Builds a visual drag handle element containing:

      • Color squares corresponding to each channel in the chart
      • Channel labels next to each color
      • Reactive subscriptions to update labels when state changes

      The drag bar serves multiple purposes:

      1. Visual identification of which channels are in each chart
      2. Drag handle for reordering charts (via setupChartDragAndDrop)
      3. Live update target when channel labels change in UI

      Internal Behavior:

      • Infers channel type (analog/digital) based on index ranges
      • Attempts to subscribe to yLabels updates via createState API
      • Handles both whole-array and single-element updates
      • Gracefully ignores subscription errors on fallback systems
      • Preserves data-global-index attributes for live label updates
    Examples
    // Create drag bar for analog chart with 3 channels
const dragBar = createDragBar(
  { indices: [0, 1, 2], colors: ['#FF0000', '#00FF00', '#0000FF'] },
  cfg,
  channelState
);

// HTML output structure:
// <div class="dragBar" draggable="true" style="display:flex;...">
//   <div class="dragBar-row" style="display:flex;...">
//     <span class="dragBar-color" style="background:#FF0000;..."></span>
//     <span class="dragBar-label" data-global-index="0">Phase A Current</span>
//   </div>
//   <div class="dragBar-row">
//     <span class="dragBar-color" style="background:#00FF00;..."></span>
//     <span class="dragBar-label" data-global-index="1">Phase B Current</span>
//   </div>
//   <div class="dragBar-row">
//     <span class="dragBar-color" style="background:#0000FF;..."></span>
//     <span class="dragBar-label" data-global-index="2">Phase C Current</span>
//   </div>
// </div>
    // The drag bar automatically updates when labels change
// Before: "Phase A Current"
channelState.analog.yLabels[0] = "Main Feeder A";
// After: Drag bar label automatically updates to "Main Feeder A"
    // Digital channel drag bar
const digitalDragBar = createDragBar(
  { indices: [0], colors: ['#FFAA00'] },
  cfg,
  channelState  // Will infer type="digital" based on index ranges
);
